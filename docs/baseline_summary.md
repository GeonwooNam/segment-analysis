## 1. 결과 요약

| | 모델A — Tier1+2 (31피처) | 모델B — 전체 (175피처) |
|---|:---:|:---:|
| **OOF Macro-F1** | **0.4512** | **0.4809** |
| A (162명, 0.04%) | 0.215 | 0.199 |
| **B (24명, 0.006%)** | **0.000** | **0.000** |
| C (21,265명, 5.3%) | 0.579 | 0.658 |

**B=0.000** → Macro-F1에서 1/5 ≈ 0.1 패널티 상시 부담.

## 2. 주요 발견

### 피처 중요도 상위 10 (모델B, gain 기준)

| 순위 | 피처 | 비고 |
|---|---|---|
| 1 | `입회경과개월수_신용_last` | 고객 연차 — 예상대로 1위 |
| 2 | `카드이용한도금액_mean_6m` | 스냅샷보다 **6m 평균**이 더 강함 |
| 3 | `이용금액_오프라인_B0M_mean_6m` | 오프라인 이용 패턴 |
| 4 | `증감율_이용금액_신용_전월_mean_6m` | **trend-like 신호** — 성과정보에서 포착 |
| 5 | `증감율_이용금액_신용_전월_std_6m` | 변동성도 중요 |
| 6 | `증감율_이용금액_신용_분기_last` | |
| 7 | `최종카드발급경과월_last` | 카드 갱신 주기 |
| 8 | `CL이자율_할인전_std_6m` | 이자율 **변동성** |
| 9 | `카드이용한도금액_std_6m` | 한도 변동성 |
| 10 | `이용금액_오프라인_B0M_std_6m` | |

**의외의 발견:**
- `증감율` 계열이 4~6위 — 직접 trend 피처 없이도 성과정보의 증감율이 강한 대리 신호. Phase 4 trend 피처 추가 시 추가 효과 기대
- `연체여부_ever`, `시장단기연체여부` = **중요도 ≈ 0** — 한도/이용금액 수준이 훨씬 강한 분리 신호
- **6m 평균/표준편차**가 단순 snapshot보다 중요한 경우 多 (2위 `mean_6m` vs 16위 `_last`)

### 중요도 = 0 제거 대상 (8개)
`시장단기연체여부_R3M/R6M`, `대표결제방법코드_last`, `컨택건수_리볼빙TM/FDS_R6M`, `당사PAY_방문횟수_R6M`, `불만제기건수_R12M`

### Early Stopping 미발동
1000트리 전부 소진 → loss가 계속 감소 중. **n_estimators=2000~3000 + lr=0.02** 재실험 여지 있음.

### 피처 수 vs 성능
- 31개 → 175개: Macro-F1 +0.030 (0.4512 → 0.4809)
- 단, A 클래스는 31개 모델(0.215)이 175개(0.199)보다 높음 → 피처 과다가 A/B 구분에 노이즈 추가 가능성

## 3. B 클래스 F1=0.000 원인 분석

| 원인 | 설명 |
|---|---|
| **절대적 샘플 부족** | 5-fold 기준 train fold당 B 고객 ≈ 19명, val에 ≈ 5명 |
| **E와 클래스 불균형** | E가 80%라 모델이 E를 기본값으로 출력해도 loss가 낮음 |
| **A·B가 C에 흡수** | 한도·이용금액 높은 고객을 C로 분류하는 경향 — A와 B 경계가 불명확 |
| **class_weight 한계** | B weight≈3333을 줘도 5명 샘플의 학습 신호가 너무 약함 |

## 4. B 클래스 잡기 — 아이디어 & 계획

### 전략 1 — 2단계 계층 모델 ⭐ (우선순위 높음)

```
Stage 1: E vs 非E (이진)
  → 전체 400k 중 E(320k) vs 非E(80k) 분류
  → E를 먼저 걷어내면 Stage 2 학습 데이터에서 B 비중 상승
     (非E 79,658명 중 B 24명 → 0.03%, but E 노이즈 제거)

Stage 2: A / B / C / D (4분류, 非E만 대상)
  → 모델이 B에 집중할 수 있는 환경
  → B weight 더 공격적으로 올릴 수 있음
```

**기대 효과:** Stage 1이 E를 잘 걷어낼수록 Stage 2의 B 검출 정밀도 상승.  
**리스크:** Stage 1에서 실제 非E를 E로 잘못 분류하면 연쇄 오류.

---

### 전략 2 — A+B 클래스 병합 후 재분리 ⭐

```
Stage 1: [AB] vs C vs D vs E (4분류)
  → A(162) + B(24) = 186명을 하나의 프리미엄 클래스로 묶음
  → 186명 vs 나머지를 학습하면 24명 단독보다 신호 훨씬 강함

Stage 2: AB로 분류된 고객 중 A vs B (이진)
  → 186명 내에서 A와 B를 구분하는 세밀한 분류기
  → A(162) vs B(24)는 여전히 불균형이나 E가 없어 훨씬 집중 가능
```

**기대 효과:** B F1을 0에서 탈출시키는 가장 현실적인 구조.  
**구현 난이도:** 중간.

---

### 전략 3 — Threshold 후처리 조정

```
현재: argmax(prob) → B 예측 확률이 제일 높아야 B
조정: B_prob > θ_B 이면 B 예측 (θ_B << 0.5)

OOF 확률값(predict_proba)으로 θ_B 탐색:
  for θ in [0.001, 0.005, 0.01, 0.02, 0.05]:
      pred = argmax + B override if B_prob > θ
      Macro-F1 계산 → 최적 θ 선택
```

**기대 효과:** 코드 변경 최소화, 빠르게 B recall 올릴 수 있음.  
**리스크:** θ 낮추면 실제 C/D 고객을 B로 오분류(precision 하락) → F1 최적점 탐색 필요.  
**적용 시점:** 어떤 모델이든 마지막에 항상 적용 가능한 후처리.

---

### 전략 4 — B 특화 파생 피처 추가

A·B 고객이 다른 세그먼트와 구별되는 극단값 피처 생성:

```python
# 초고한도 플래그 (상위 1%)
df['초고한도_flag'] = (df['카드이용한도금액_last'] > df['카드이용한도금액_last'].quantile(0.99)).astype(int)

# 한도 × 이용금액 (VIP 복합 지수)
df['vip_score'] = df['카드이용한도금액_last'] * df['이용금액_신용합계_B0M_mean_6m']

# 이자율 극단 저금리 플래그 (최우량 신용)
df['극저금리_flag'] = (df['CL이자율_할인전_last'] < df['CL이자율_할인전_last'].quantile(0.05)).astype(int)

# 장기 무연체 플래그
df['장기무연체_flag'] = ((df['입회경과개월수_신용_last'] > 60) & (df['회원여부_연체_ever'] == 0)).astype(int)
```

**기대 효과:** B/A 판별에 직접적인 시그널. 트리 모델에서 분기를 단순화.

### 전략 5 — sample_weight 극단 조정

```python
# 현재: 빈도역수 (B weight ≈ 3333)
# 실험: B weight를 10,000 / 30,000 / 100,000 으로 올려보기

# 주의: 너무 높이면 B 5명에 과적합 → val에서 E/C를 B로 오분류
# CV로 최적 multiplier 탐색
for b_mult in [1, 3, 10, 30]:
    weight = {0: w_A, 1: w_B * b_mult, 2: w_C, 3: w_D, 4: w_E}
```

**기대 효과:** 구현 가장 간단. 하지만 효과 한계 있음(근본 샘플 부족 해결 불가).  
**전략 3(threshold)과 병행** 시 효과 극대화.

### 전략 6 — KNN 기반 B 보조 탐지기

```
B 24명의 피처 벡터로 KNN 학습
→ Test 고객 중 B 이웃이 K명 이상이면 B 후보 플래그
→ 메인 모델 예측 + KNN B 플래그를 앙상블 규칙으로 결합

예) main_pred != B AND knn_B_score > threshold → override to B
```

**기대 효과:** 비모수적 접근, 24명의 특성 공간을 직접 활용.  
**리스크:** 차원의 저주 (175차원에서 거리가 의미 없을 수 있음) → PCA 축소 후 적용 권장.

---

### 전략 7 — n_estimators 증가 + 파라미터 재조정

```
현재: n_estimators=1000, lr=0.05, min_child_samples=5
개선안: n_estimators=3000, lr=0.02, min_child_samples=1

early stopping이 미발동 = 아직 학습 여지 있음
min_child_samples=1: leaf 하나에 B 1명만 있어도 분기 허용
```

**기대 효과:** 전체 Macro-F1 0.48 → 0.50대 진입 가능.  
**실행 난이도:** 낮음 (파라미터 변경만).
