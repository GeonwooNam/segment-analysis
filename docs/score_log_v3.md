## ⚠ OOF 0.8361 과 Public 0.5926 을 직접 비교하지 말 것

`model_v3_base.py`가 출력하는 `OOF macro-F1 — ID 단위`는 **C/D/E 3개 클래스**만의 macro-F1이다.
(base 모델은 학습 전에 A/B를 제거하므로 A/B를 아예 모른다.)

    (0.7716 + 0.7667 + 0.9700) / 3 = 0.8361     <- base가 출력하는 값
    (0.7716 + 0.7667 + 0.9700) / 5 = 0.5017     <- A=B=0 일 때의 5-class macro

리더보드는 **5개 클래스** macro-F1이다. Public 0.5926 이므로:

    A, B 두 클래스의 기여 합 = 0.5926 * 5 - 2.5083 = 0.455  (만점 2.000)

즉 C/D/E 부분은 OOF 예측과 거의 일치하며 **CV 붕괴가 아니다.**
가이드 7절의 "갭 ±0.02 이상이면 CV 재점검"은 서로 다른 지표를 비교한 것이라 이 케이스에 적용되지 않는다.
앞으로는 combine 로직을 OOF에 그대로 적용한 **5-class ID 단위 macro-F1**을 기록할 것.

## VIP_CAP 은 이미 최적 — 첫 튜닝 포인트가 아니다

OOF(train, 400k ID)에서 cap 스윕. test cap 60 == train cap 240.

| cap_train | ~cap_test | precision | recall | AB F1 |
|---|---|---|---|---|
| 120 | 30 | 0.317 | 0.204 | 0.248 |
| 186 | 46 | 0.280 | 0.280 | 0.280 |
| **240** | **60** | **0.250** | **0.323** | **0.282** |
| 400 | 100 | 0.203 | 0.435 | 0.276 |
| 900 | 225 | 0.123 | 0.597 | 0.204 |

최적은 cap_test≈65 에서 AB F1 0.291. 현재 60에서 0.282.
곡선이 평평해서 cap 튜닝의 이득은 macro-F1 기준 ~0.002 수준 — 무의미하다.
