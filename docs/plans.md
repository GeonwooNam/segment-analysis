## Phase 2 — 피처 엔지니어링 & 모델링 테이블 생성

**목표**: 고객 단위(40만 행) 마스터 모델링 테이블 구축  
**방법**: 6개월 × 8테이블 → 고객당 1행으로 집계

### 집계 방식 (컬럼 유형별)
| 유형 | 처리 |
|---|---|
| 스냅샷성 (한도, 잔액) | 마지막달(201812) 값 + R6M mean |
| 누적 활동 (이용금액·건수) | 6개월 sum + mean + std + 추세(기울기) |
| 최근성 (최종이용일자) | sentinel 제거 후 기준월 대비 경과월(recency) |
| 범주형 (연령, 성별) | 마지막달 값(불변이면 그대로) |

### 결측·sentinel 처리 확정
- 날짜 sentinel `00010101`/`10101` → recency 컬럼 + `_이용없음` 플래그
- 금액/건수 `0` → 그대로 (진짜 무활동)
- MNAR 계열(`_N순위…`, `최종카드론…`) → 결측 플래그 컬럼 추가, 값은 NaN 유지(트리 네이티브)

### 산출물
- [x] **P2-1** 고객 단위 집계 함수 + 마스터 테이블 생성 스크립트 (`build_features.py`, 176컬럼)
- [x] **P2-2** train/test 동일 처리 파이프라인 (`features/train_features.parquet` 400k×176, `test_features.parquet` 100k×175)

## Phase 3 — 베이스라인 모델링

**전략**
- 모델: **LightGBM** (범주형·NaN 네이티브, 속도)
- 불균형: `class_weight` (빈도역수) — 리샘플링 없이 먼저 시도
- CV: 고객(ID) 단위 **StratifiedKFold 5fold**, Segment stratify
- **리샘플링/증강은 train fold에만**, validation은 원본 분포 그대로
- 지표: **Macro-F1**
