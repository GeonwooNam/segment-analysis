## 1. 데이터 구조

| 항목 | 내용 |
|---|---|
| 테이블 수 | 8개 (회원/신용/승인매출/청구입금/잔액/채널/마케팅/성과) |
| 조인 구조 | `(ID, 기준년월)` **1:1 조인**, 월별 40만 행, 중복 0 |
| 패널 구조 | train 40만 고객 × 6개월, **완전 균형 패널** |
| **예측 단위** | **고객(ID) 1명 = 라벨 1개** — Segment가 6개월 내내 불변 |

**핵심**: 트랜잭션 레벨이 아니라 이미 고객·월 단위로 집계된 와이드 테이블. 8개 테이블을 `(ID, 기준년월)`으로 flat join하면 끝.

## 2. 타깃 (Segment) 분포

| Segment | 고객 수 (train) | 비율 |
|:---:|---:|---:|
| A | 162 | 0.04% |
| **B** | **24** | **0.006%** |
| C | 21,265 | 5.3% |
| D | 58,207 | 14.6% |
| E | 320,342 | 80.1% |

- **A·B = 프리미엄 고객**, E = 일반/저활동 고객 (한도 단조 감소: A>B>C>D>E)
- 평가지표 **Macro-F1** → B(24명)가 전체 점수의 **1/5**을 차지. E만 잘 맞히면 점수 낮음
- B 24명은 훈련셋 전체에서 가장 작은 클래스 — 소수 클래스 정밀도/재현율이 점수의 핵심

## 3. 결측치 & Sentinel 패턴

### 진짜 NaN (MNAR — 결측 자체가 신호)
| 컬럼 | 결측률 | 의미 |
|---|---:|---|
| `_3순위여유업종` 등 N순위 계열 | 84~98% | 해당 업종 이용 없음 = 저활동 고객 |
| `최종카드론_대출일자` | 83% | 카드론 미이용 |
| `_2순위신용체크구분` | 39% | 카드 2장 미소지 |
| `최종유효년월_신용_이용` | 20% | 미이용 |
| `가입통신회사코드` | 17% | 정보 없음 |

→ **임퓨트하지 말고** 결측 플래그 컬럼 추가 + NaN 유지 (트리 모델 네이티브 처리)

### Sentinel (위장 결측)
- **날짜 `10101`** (= 0001-01-01): 미이용 표시. 예) `최종이용일자_할부` 28%가 10101
  - 처리: `_이용없음` 이진 플래그 생성 + 실제 날짜는 recency(경과월)로 변환
- **금액/건수 `0`**: 진짜 무활동. 그대로 유지 (LightGBM이 분기에서 학습)

## 4. 테이블별 주요 발견

### 2. 신용정보 (분리력 ★★★★★)
- `카드이용한도금액`: Segment 간 **가장 강한 분리력**. A·B는 한도 수천만 원대, E는 수백만 원대 → 단일 최강 피처
- `CA이자율_할인전`, `CL이자율_할인전`: 상위 세그먼트일수록 이자율 낮음 (신용도 반영)
- `RV약정청구율`: E 고객에서 높음 (리볼빙 부채 성향)
- `_1순위신용체크구분` (회원정보에 존재): 신용카드만 vs 신용+체크 혼용 패턴 차이

### 5. 잔액정보 (분리력 ★★★★☆)
- `총잔액` (일시불+할부+CA+카드론 파생): A·B >> C·D·E. log 스케일에서도 명확히 분리
- `연체잔액_B0M`: D·E 고객에서 높음. 연체여부 이진 플래그가 강한 시그널
- `잔액_카드론_B0M`: 상위 세그먼트에서 카드론 잔액 높음 (고신용 한도 활용)

### 3. 승인매출정보 (분리력 ★★★★☆, 406컬럼)
- `이용금액_신용합계_B0M` (일시불+할부+CA 파생): A·B >> E. 이용금액이 Segment를 가름
- `이용후경과월_신용`: E > D > C > B > A. E 고객일수록 마지막 이용이 오래됨 (미이용 고객)
- `이용없음_기본` 플래그: E 고객의 상당수가 당월 미이용 (sentinel 10101)
- 406컬럼 중 B0M·R3M·R6M 이용금액/건수 계열이 핵심, 나머지 대부분 파생·세부업종

### 8. 성과정보 (분리력 ★★★☆☆)
- `잔액_신판평균한도소진율_r6m`: 한도 대비 이용 비율 — 상위 세그먼트가 한도를 더 활용
- `증감율_이용금액_신용_전월`: 이용 성장 트렌드 반영, 성장 중인 고객이 상위 세그먼트 경향
- `혜택수혜율_R3M`: 상위 세그먼트일수록 혜택(포인트·할인) 활용도 높음
- float 컬럼 多 → NaN 처리 주의

### 1. 회원정보 추가 (분리력 ★★★☆☆)
- `입회경과개월수_신용`: 상위 세그먼트 = 오래된 고객. 장기 우량 고객 패턴
- `회원여부_연체` (1/0): 연체=1이면 D·E 집중. 단순하지만 강한 시그널
- `이용거절여부_카드론`: 하위 세그먼트에서 거절 높음
- `소지카드수_이용가능_신용`: 이용 가능 카드 수 차이

### 4. 청구입금정보 (분리력 ★★★☆☆)
- `청구금액_R6M`: 6개월 누적 청구금액. 이용금액과 강하게 연동, A·B >> E
- `대표결제방법코드`: 결제 방법(자동이체·일시불 등)으로 세그먼트 패턴 차이
- `포인트_마일리지_건별_B0M`: 상위 세그먼트 포인트 적립 활발
- 결측 0개 — 가장 클린한 테이블

### 6. 채널정보 (분리력 ★★☆☆☆)
- `방문횟수_앱_R6M`, `방문횟수_PC_R6M`: 디지털 채널 활용도. 상위 세그먼트일수록 앱/PC 적극 활용
- `인입횟수_ARS_R6M`: 하위 세그먼트(연체·문의)에서 높을 가능성 (고객센터 콜)
- `채널활동성_합산` (파생): ARS+PC+앱+모바일웹 합계 — 전반적 디지털 참여도

### 7. 마케팅정보 (분리력 ★★☆☆☆)
- `컨택건수_카드론_TM_B0M`: 카드론 TM = 하위 세그먼트 집중 타겟
- `컨택건수_이용유도_TM_B0M`: 저활동(E) 이용 유도 집중
- `캠페인접촉건수_R12M`: 마케팅 타겟팅이 집중된 세그먼트 식별
- 전반적으로 분리력 낮음 — Tier 2/3 보조로만 활용

## 6. A·B 프리미엄 고객 식별 단서

B(24명), A(162명)를 찾아내는 게 Macro-F1의 핵심.

- **초고한도** (`카드이용한도금액` 상위 꼬리) — 가장 강한 시그널
- **고이용금액** (`이용금액_신용합계_B0M`, `청구금액_R6M` 상위)
- **장기 우량**: `입회경과개월수_신용` 길고 `회원여부_연체=0`
- **디지털 활성**: `방문횟수_앱_R6M` 높음
- **낮은 이자율**: `CA이자율_할인전`, `CL이자율_할인전` 낮음 (신용도 우수)
- **마케팅 미타겟**: 카드론/이용유도 TM 컨택 없음 (이미 우량이라 타겟 불필요)
