## 개요

**Phase 2 = Feature Engineering (피처 엔지니어링)**  
원본 데이터(6개월 × 8테이블 × ~872컬럼)를 **고객 1명당 1행**으로 변환한 모델 입력 테이블 생성.  
※ Feature Selection(중요도 낮은 컬럼 제거)은 Phase 3 베이스라인 모델 이후에 수행.

---

## 집계 전략

| 컬럼 유형 | 집계 방식 | suffix | 예시 |
|---|---|---|---|
| 스냅샷성 (한도·상태·범주) | 201812 마지막달 값 | `_last` | `카드이용한도금액_last` |
| 누적 활동 (이용금액·건수) | 6개월 mean + std | `_mean_6m`, `_std_6m` | `이용금액_B0M_mean_6m` |
| 이진 상태 (연체·ever) | 6개월 max — 한 번이라도 발생 시 1 | `_ever` | `회원여부_연체_ever` |
| Sentinel 날짜 | 미이용 플래그 + recency 변환 | `_이용없음` | `이용없음_기본` |
| MNAR 결측 | 결측 여부 이진 플래그 + NaN 유지 | `_결측` | `_2순위신용체크구분_결측` |

**Trend(기울기)는 1차 구현 제외** — 컬럼 수 폭발 대비 효과 불확실. mean/std로 대체.

---

## Sentinel & 결측 처리 규칙

### 날짜 Sentinel (`10101` = 0001-01-01 = 미이용)
```
최종이용일자_기본 등의 10101 값:
  → 이용없음_기본 = 1  (이진 플래그)
  → 이용후경과월  = 999 (recency, E 고객 특징)
비-sentinel:
  → 경과월 = 201812 - (날짜 // 100)  ex) 20180715 → 201807 → 경과 5개월
```

### MNAR 결측 (결측 = 미이용·저활동 신호)
```
_2순위신용체크구분 (결측 39%): 카드 2장 미소지
가입통신회사코드   (결측 17%): 정보 없음
직장시도명        (결측 10%): 미취업 or 미제공
  → 각각 _결측 플래그 + 원본 NaN 유지 (LightGBM 네이티브)
```

## 주의사항 (모델링 시)

1. **채널정보 범주형 string**: `방문횟수_앱_R6M_last` 등 — `pd.Categorical` or LightGBM `categorical_feature` 지정 필요
2. **`OS구분코드_last` 결측 68%**: 제거 대신 유지 (미앱이용 = E 고객 특징)
3. **날짜 컬럼 `대표결제일_last`**: yyyymmdd 형태 int — 모델 투입 전 recency 변환 권장 (또는 그대로 트리에 투입)
4. **train Segment 불균형**: E 80% / B 24명 → Phase 3에서 `class_weight` 빈도역수로 처리
